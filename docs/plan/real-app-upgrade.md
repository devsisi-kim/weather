# Real App Upgrade Notes

## 목표
- MVP(클라이언트 중심)에서 실제 운영 가능한 구조(백엔드 API + 서버 저장 + 테스트)로 전환.

## 적용 사항
1. 백엔드 서버 추가
- Node HTTP 서버 기반 API 구현
- 정적 파일 서빙 + API 라우팅 분리

2. 서버 데이터 저장
- `data/locations.json`에 위치 목록 저장
- 서버에서 최대 2개 제한 강제

3. 서버 추천 생성
- 서버가 날씨 수집 후 추천 로직 적용
- 클라이언트는 렌더링 전담

4. 운영성
- `/api/health` 제공
- 입력/오류 메시지 표준 JSON 응답

- [x] 추천 결과를 서버에서 생성
