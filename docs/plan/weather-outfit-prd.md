# PRD - Weather Outfit Recommender (MVP)

## 1. 문제 정의
- 배경: 사용자는 "오늘 뭐 입지"를 빠르게 판단하고 싶다.
- 현재 문제: 기온만 보고 판단하면 UV/습도/비를 놓치기 쉽다.
- 해결 목표: 저장된 1~2개 위치의 당일 추천 복장/소품을 한 화면에서 제시한다.

## 2. 사용자/시스템 시나리오
- 주요 사용자: 출근/외출 전에 빠르게 옷차림을 정하는 일반 사용자
- 핵심 시나리오:
  1) 위치를 최대 2개 저장
  2) 앱이 현재 날씨 지표를 조회
  3) 추천 옷 + 소품 + 이유를 확인

## 3. 인수 기준 (Acceptance Criteria)
- [ ] AC-1: 사용자는 위치를 1~2개 저장/삭제할 수 있다.
- [ ] AC-2: 각 위치별 현재 기온, 습도, UV index, 강수 확률을 표시한다.
- [ ] AC-3: 규칙 엔진이 옷 추천(대표 이미지 1개 + 텍스트)을 반환한다.
- [ ] AC-4: 소품 추천(모자/선크림/우산 등)이 날씨 지표에 따라 달라진다.
- [ ] AC-5: 데이터 조회 실패 시 사용자에게 오류 메시지를 보여준다.

## 4. 제약사항 (Constraints)
- 기술 제약: 서버 없이 브라우저 기반 MVP
- 일정/리소스 제약: 단일 화면, 단순 규칙 엔진
- 성능/비용 제약: 무료 API 사용

## 5. 범위 제외 (Out of Scope)
- 장기 예보 기반 추천
- 사용자 계정/동기화
- AI 기반 개인 맞춤 스타일링

## 6. 성공 지표 (Success Metrics)
- 정량 KPI: 최초 로딩 후 3초 내 추천 표시(정상 네트워크 기준)
- 품질 KPI: 규칙 로직 단위 테스트 통과율 100%

## 7. 리스크 및 대응
- 리스크: 외부 API 실패/지연
- 대응 전략: 오류 메시지 및 재시도 버튼 제공

## 8. 릴리스 전략
- 배포 단위: 정적 웹앱
- 롤백 기준: 추천 로직 오류로 잘못된 계절군 제안이 반복되는 경우
