# Research Brief - Weather Outfit Recommender

## 1. 문제/아이디어 요약
- 한 줄 정의: 저장된 1~2개 위치의 당일 날씨를 보고, 옷과 소품 추천을 빠르게 받는 경량 앱.
- 가설: 체감 핵심 지표(기온, 습도, UV, 강수 가능성)를 함께 반영하면 단순 "기온별 옷차림"보다 실사용 만족도가 높다.

## 2. 기존 사례 조사
- 사례 1: 일반 날씨 앱의 "오늘 날씨" 카드
  - 장점: 데이터 접근성 높음
  - 한계: 옷 추천이 약하거나 기온 중심 단순 규칙
- 사례 2: 기온별 옷차림 인포그래픽(사용자 첨부 이미지 유형)
  - 장점: 직관적이고 빠름
  - 한계: 습도/UV/강수 반영이 제한적
- 사례 3: Open-Meteo API 조합형 앱
  - 장점: 무료/키 불필요, 지표 다양
  - 한계: 추천 규칙은 앱 개발자가 별도 설계 필요

## 3. 대안 비교
- 대안 A: 기온 단일 기준 추천
  - 구현이 가장 빠름
  - 정확도/상황 대응(자외선, 비)이 낮음
- 대안 B: 기온+습도+UV+강수 결합 규칙 추천
  - 구현 난이도 소폭 증가
  - 설명 가능성(규칙 기반)과 만족도 균형이 좋음
- 비교 요약: MVP에서는 대안 B가 적정.

## 4. 평가
- 기대 가치: 외출 전 의사결정 시간을 줄여줌
- 구현 복잡도: 낮음~중간 (정적 프론트 + 외부 API 호출 + 규칙 엔진)
- 주요 리스크: 지역명 지오코딩 실패, 기상 급변 시 추천 오차
- 불확실성: 개인 체감 온도 편차(개인화 미포함)

## 5. 권고안
- 지금 할 것:
  - 위치 1~2개 저장
  - 당일 지표 조회(기온/습도/UV/강수)
  - 규칙 기반 옷/소품 추천 + 대표 이미지
- 나중에 할 것:
  - 사용자 체감 프로필(더위/추위 민감도)
  - 시간대별 추천
- 하지 않을 것:
  - 계정/로그인
  - 복잡한 패션 코디 엔진

## 참고 출처 검토
1) Open-Meteo 공식 문서
- API 필드 정의와 예제가 명확하고, 제품 제작에 직접 쓰는 1차 자료(공식 문서)라 신뢰도 높음.
- 단, 무료 API 정책/가용성은 변경 가능하므로 배포 전 재확인 필요.

2) World Health Organization (WHO) - UV index 안내
- 공공보건 기관 자료로 UV 위험 구간 해석 근거로 신뢰도 높음.
- 단, 지역/피부 타입/노출 시간에 따라 개인 편차가 크므로 앱 규칙은 보수적으로 적용 필요.
